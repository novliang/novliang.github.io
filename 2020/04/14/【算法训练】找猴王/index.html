【算法训练】找猴王

2020-04-14

今天来看一个面试算法题，找猴王,题目如下

每次写算法题目的时候，不能单单写出来了，还要分析一下原因和时间复杂度

分析

首先是n个猴子，然后报数m，第一时间想用循环链+计数器实现，这样计数器到m，就吧当前元素的next指针指向next.next 如果next.next == 自己的话 那自己就是猴王了,但是还要实现一个循环链表，无疑增加了复杂度，就想看看有没有什么暴力的办法

数组算法

这种数组的方法应该是最简单的方法了，时间复杂度为O(m*n)，后来在网上看到大家说逆推法，顿时惊为天人，感觉自己的总结归纳和寻找的能力差太多

逆推法

动笔算了一段时间才大致弄明白这个逆推思想

例如我们有n个猴子0,1,2,3,4,...,n-1

我们让他们倒数m,则第一个被淘汰的猴子k = (m-1)%n

这时候剩下的猴子变成 0,1,2,3,4,...,k-1,k+1,k+2,k+3,...,n-1

接下来从k+1个猴子开始数m，实际猴子变成k+1,k+2,...,n-1,0,1,2,3,4,...,k-1

我们把k+1开始用一个公式转换成从编号0开始的0,2,3,4,5,...,n-2的顺序

得到公式f(x) = (x-k-1)%n 则猴子可以变成0,1,2,......,n-2的顺序

上面的公式要注意亮点，一个激素到了n-1然后从0开始之后，0-k-1得到的是个负数，负数的余数运算n = k*q + r，所以最后一个数k-1的转换结果为(k-1-k-1)%n = -2%n = 0*n + (n-2)为n-2

如果用0,1,2,3,4,....n-2逆推为k+1,k+2,.....n-1,0,1,2,3,4,.....,k-1

得到公式f(x) = (x+k+1)%n 还记不记得k = (m-1)%n 所以f(x) = (x+m)%n

算到这里可能有点头晕眼花，但是其实如果x= 0 n=1的时候也就是(0+m)/1的时候，说倒数第二轮比较的时候,例如m = 3的时候0+3% 1 = 2，然后那一轮的时候k 说第3(2+1)编号，就这样一直往前推算被选出的猴王的的k值，直到算出最一开始的时候的k值，然后加一就是当时猴王的编号了
