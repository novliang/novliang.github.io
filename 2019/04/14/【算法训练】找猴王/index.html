【算法训练】找猴王

2019-04-14

今天来看一个面试算法题，找猴王,题目如下

每次写算法题目的时候，不能单单写出来了，还要分析一下原因和时间复杂度

分析

首先是n个猴子，然后报数m，第一时间想用循环链+计数器实现，这样计数器到m，就吧当前元素的next指针指向next.next 如果next.next == 自己的话 那自己就是猴王了,但是还要实现一个循环链表，无疑增加了复杂度，就想看看有没有什么暴力的办法

数组算法

这种数组的方法应该是最简单的方法了，时间复杂度为O(m*n)，后来在网上看到大家说逆推法，顿时惊为天人，感觉自己的总结归纳和寻找的能力差太多

逆推法

以下内容经过重新编辑，后来回来复习的时候，感觉当时写的不是很清楚

有n个猴子0,1,2,3,4,...,n-1

我们让他们报数m,则第一个被淘汰的猴子k = (m-1)%n

这时候剩下的猴子变成 0,1,2,3,4,...,k-1,k+1,k+2,k+3,...,n-1

接下来从k+1个猴子开始报数m，实际猴子变成k+1,k+2,...,n-1,0,1,2,3,4,...,k-1

我们把k+1开始用一个公式转换成从编号0开始的0,1,2,3,4,5,...,n-2的顺序

得到公式f(x) = (x-k-1)%n 则猴子可以变成0,1,2,......,n-2的顺序

上面的公式要注意亮点，到了n-1然后从0开始之后，0-k-1得到的是个负数，负数的余数运算n = k*q + r(这里的k和n和上面的公式没关系，就是一个求余数的公式)，所以最后一个数k-1的转换结果为(k-1-k-1)%n = -2%n = 0*n + (n-2)为n-2

如果用0,1,2,3,4,....n-2逆推为k+1,k+2,.....n-1,0,1,2,3,4,.....,k-1

得到公式f(x) = (x+k+1)%n 还记不记得k = (m-1)%n 所以f(x) = (x+m)%n

但是其实如果x= 0 n=1的时候也就是(0+m)/1的时候，例如报数的m = 3 则 (0+3)%1 = 1 说明猴王在倒数第二轮只剩两只猴子的时候，所在的下标为1 编号为1+1 = 2然后再次倒退下标为1猴王在倒数第三轮的位置为(1+3)%2 = 0,以此类推后，直到推送到n 等于实际猴子的个数时，此时猴王的下标就是在一开始猴王所在的位置
